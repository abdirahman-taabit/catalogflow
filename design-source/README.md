# CatalogFlow Figma capture sources

These self-contained design fragments are Phase 1 artifacts used only to create editable layers in Figma. They are not the production Next.js application.

- `foundations.body.html` — tokens, typography, spacing, elevation, states, and breakpoints
- `components.body.html` — reusable UI patterns and important variants
- `desktop.body.html` — landing, dashboard, catalog, import, product review, activity, and system states
- `mobile.body.html` — 768 px tablet and 390 px mobile adaptations
- `flow.body.html` — primary flow, recovery paths, and state transitions

The visual direction uses a graphite navigation shell, warm stone canvas, white work surfaces, and restrained moss status/action color. Product enrichment is described as deterministic and rule-based throughout.
